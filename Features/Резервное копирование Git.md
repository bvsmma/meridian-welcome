# Резервное копирование Git

Meridian автоматически сохраняет и синхронизирует ваше хранилище с приватным репозиторием на GitHub. Каждое сохранение становится снимком в вашей истории версий.

---

## Настройка

1. Откройте **Настройки → Синхронизация и GitHub** (`⌘,`)
2. Подключите свой аккаунт GitHub (откроется окно в браузере)
3. Откройте панель **Резервное копирование** (иконка Git на боковой панели)
4. Нажмите **Настроить резервное копирование** и введите URL вашего репозитория
5. Готово — теперь ваше хранилище надежно сохранено

---

## Как это работает

**Сохранить снимок** — вручную зафиксировать (commit) все изменения с сопроводительным сообщением.

**Автоматический бекап** — настройте интервал (15 мин, 30 мин, 1 час) на панели. Meridian будет автоматически делать коммиты и отправлять их в облако.

**История** — каждый коммит представляет собой снимок на определенный момент времени. Просматривайте историю версий прямо на панели.

---

## Сообщения коммитов

Meridian генерирует сообщения коммитов автоматически:

```
Обновлено: Научная заметка.md, Планирование проекта.md
Добавлено: Daily/2026-05-22.md
```

Вы можете настроить шаблон сообщения в разделе **Настройки → Синхронизация → Шаблон сообщения коммита**.

---

## Почему именно GitHub?

- Бесплатно для приватных репозиториев
- Надежная синхронизация между устройствами
- Хранение в виде обычного текста — никакого проприетарного формата

> [!NOTE]
> Сначала включите плагин «Автосохранение в Git» в разделе **Настройки → Основные плагины**.

> [!TIP]
> Установите автобекап на 15 минут и забудьте о нем. Ваши заметки всегда в безопасности.
